How it works

Two tools. One closed loop.

An MCP server that plugs into Claude Code, Cursor, or any MCP-compatible tool and enforces task completeness on every task — automatically.

01 trace_dependencies

Map before you touch

Before Claude edits any file, United Agents scans your codebase and maps every connected file — components, API routes, database layers, parent files. Claude sees the full picture first.

→ call before editing

02 verify_completeness

Block premature "done"

Before Claude says done, United Agents checks what was actually changed against the dependency map. Missing a file? Claude keeps working automatically — no asking, no waiting.

→ call before saying done

03 setup_project

Set and forget

Writes a CLAUDE.md into your project enforcing the trace → execute → verify loop on every task. Claude reads it at session start. Run once. Always active.

→ run once per project

Install

Two commands.
Never think about it again.

Install once globally. Set up each project once. After that — open Claude Code normally. United Agents runs invisibly every time.

Step 01 — Install globally, once ever
$ npm install -g united-agents-mcp copy

Installs the MCP server globally on your machine. Never run again.

Step 02 — Set up each project, once
$ united-agents-mcp setup copy

Run from your project folder. Registers globally and writes CLAUDE.md. Never run again for that project.
